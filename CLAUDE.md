# 競馬予想PWAサイト — Claude用 更新手順書

このリポジトリは GitHub Pages で公開している競馬予想PWA（スマホでホーム画面追加・オフライン可）。
**スマホ/PCのどこからでも、Claudeがこの手順に従って予想を更新→pushすれば、スマホアプリに自動反映される。**

公開URL: https://furuta-gmo.github.io/keiba-yosou/

## 方針（不変）
- **回収率重視・期待値型**。EV = 想定勝率 × 単勝オッズ。EV>1.0 が買い、妙味（人気と能力のズレ）を狙う。
- 圧倒的人気馬の**単勝は買わない**（過剰人気）。複勝率が高ければ連系の「軸」としてのみ使う。
- 買い目は**100円単位**、合計は予算ちょうど（既定10,000円）。
- **回収床**：主要ペアが1-2着なら馬連＋ワイドで概ね予算以上を返す設計。3連複は妙味・上振れ用。
- **直前の馬場急変は最優先で反映**：稍重〜重なら重巧者（例: ゴールドシップ産駒＝メイショウタバル型）を格上げ、良馬場巧者・切れ味型を割引。

## 更新の鉄則（必ず守る）
1. 予想本体は `2026-06-14_takarazuka/prediction.html` を編集（新レースは新フォルダを作り index.html に行を追加）。
2. **必ず `sw.js` の `CACHE` バージョンを上げる**（例 `keiba-yosou-v9` → `v10`）。これを忘れるとスマホで古い表示が残る。新規ファイルは `sw.js` の `ASSETS` にパスを追記。
3. 各HTMLの `<head>` には PWAスニペット（theme-color / apple-mobile-web-app-* / `<link rel="manifest" href="../manifest.webmanifest">` / apple-touch-icon / `navigator.serviceWorker.register('../sw.js')`）が必要。レース直下は `../`、analysis配下は `../../`、ルートは相対なし。新規ページには必ず付与。
4. 文字コードは UTF-8（BOMなし）。
5. 編集後: `git add -A && git commit -m "..." && git push`。GitHub Pagesが自動再ビルド（~1分）→スマホ反映。
6. バージョン表記（prediction.htmlの `vN`、index.htmlの「最終予想 vN」）も更新。
7. **反映は必ず `main` まで（ユーザー承認済・確認不要）**：GitHub Pagesは `main` から配信されるので、作業ブランチに留めるとスマホに反映されない。作業ブランチで作業した場合も、**確認なしで `main` にマージ＆push して反映する**（fast-forward優先、コンフリクト時のみ相談）。push後にスマホ反映されない時は #5 と「SWはオンラインで2回開くと更新」を案内。

## 最新情報を渡されたときの動き方
- **オッズ更新**：`odds.html` の BASE配列の既定オッズも最新に更新。prediction.html の期待値表・買い目を再計算（EV=想定勝率×単勝）。妙味馬を相手に厚く、回収床を維持。
- **馬体重/追い切り/枠**：該当馬の評価を上げ下げし、prediction.html とresearch.html に反映。
- **馬場急変（最優先）**：稍重以上なら重巧者を◎/○へ格上げ、良馬場前提の馬を割引。買い目の比重を即組み替え。
- **タイミング型モジュール（事前→枠順後→当日の3段更新）**：
  - **⑨枠順後**（枠確定＝木曜頃に「○○ 枠順 …」が来たら）：`analysis/9_枠順バイアス.html` を更新。不利枠の人気馬を割引／好枠の妙味馬を格上げ。買い目を**馬名→馬番**に確定。
  - **⑩当日**（「○○ 当日：馬場=／馬体重=／パドック=／最終オッズ」が来たら）：`analysis/10_当日.html` を更新。**確定馬場で軸を最終決定**→馬体重・気配で取捨微修正→当日バイアスで先行/差し比重→最終オッズでEV確定。prediction.html の印・買い目も最終化。
  - 枠/当日が未確定の段階では、両ページは「既知のコース傾向＋判断ルール＋未確定明記」の待ち受け型に保つ（捏造しない）。
- 想定勝率モデルの目安は `2026-06-14_takarazuka/odds.html` の BASE配列（p=想定勝率%）。

## 9. 予想精度の改善ルール（2026-06-21 府中牝馬S・しらさぎS 大惨敗の反省＋ディープリサーチを反映）
**敗因＝過更新・データ細部偏重・軸1頭依存・-EV軸。以下を必ず守る（検証ページ: track.html）。**
1. **EVは「市場ブレンドp」で計算する**（最重要）。最終p = `w×自分のp +(1-w)×市場インプライド確率`（市場p = (1/単勝)÷Σ(1/単勝)で控除率調整）。**w=0.3〜0.5**。自分のpの過信を市場が矯正し、EVが「市場に対する本当のエッジ」だけを拾う。odds.html はこのブレンド式でEVを出す（track.html に実装済の参照計算あり）。
2. **EV<1.0の馬は軸・単勝にしない**（機械的）。適性や血統で正当化して-EVの馬を軸にしない（しらさぎ◎ファーヴェントEV0.90を先行適性で軸にした過ち）。
3. **更新ロック**：一度確定した◎/▲は「事前に定めた条件（馬場の階級変化・出走取消・馬体重±15kg級・重大乗替）」が起きた時だけ動かす。**オッズ変動だけでは印を動かさない**（ブレンドpが自動で吸収）。各更新に「何が変わって、なぜ動かすか」を1行残す。府中セキトバを早期▲穴(EV2.0)→短縮のたびに格下げ→1着、の過ちを防ぐ。
4. **地力を主軸に、薄サンプルのデータ傾向を過信しない**。重賞実績・JRAレーティング・相手関係を最重視。「斤量別[0-2-0]」等の施行数が少ない傾向で有力馬を消さない（しらさぎでエルトン=マイルCS級/キープ=昨年覇者を切り1-2着の過ち）。**3連系の相手には実績上位馬を必ず残す（地力カバー）**。
5. **軸1頭依存を避ける**：上位2〜3頭をそれぞれ軸にしたポートフォリオ的買い（結合最適化の簡易版）。軸が飛んで全滅・人気薄激走を取りこぼす事態を構造的に回避。
6. **資金配分は fractional Kelly（half-Kelly目安）**。フルKellyは推定誤差で過剰賭け→破産。ドローダウン上限も意識。
7. **較正の事後検証**：予想ごとにp・結果を track.html に蓄積し、Brier/較正曲線/回収率で自分のpの癖（過信/過小）を長期で矯正。**サンプルが貯まるまで巧拙を判定しない（運と実力の分離）**。
8. **注意（誤解の是正）**：「人気薄を避けろ(favorite-longshot bias)」は欧米の話で日本は反証あり＝JRA自前データで券種別に検証してから使う。終値オッズも完全較正ではない＝市場を絶対視しない。クッション値はJRA公式の客観馬場硬度指標として④に入力可（予測寄与は未実証ゆえ過信しない）。
9. **G1の市場較正 検証結果（track.htmlのG1/25セクション・2025通年23＋2026の12）**：2026は1番人気が勝率58%と異常に強かったが**小標本の偏り**で、**2025通年は勝率30%・複勝61%・単勝回収率80%＝長期定説どおり（市場は効率的＝1番人気を買うだけでは控除率分マイナス）**。よって「人気馬を妄信しない」基本姿勢は正しく、府中・しらさぎの敗因は「人気馬を切ったこと」ではない（両レースとも5人/7人気が勝つ通常パターン）。**真因は"どの非人気馬を選ぶか"を誤り、流れた幻の妙味に賭けて地力上位の中穴を切ったこと**。blendの本質は「人気馬を持ち上げる」でなく「薄い根拠で動かした自分のpを市場較正で引き戻す」こと。穴は"勝つ"より"2-3着で絡む"＋3連単分散が極端＝地力カバー＋複数軸必須。**外部G1で検証できるのは市場較正までで、自分のpが市場に勝てるかは毎レース予想を track.html に前向きに記録(forward-test)して初めて分かる＝サンプル蓄積が精度向上の本筋。**

## 10. 大妙味G1を厚く取る戦略（ディープリサーチ反映・ツール: bet.html）
**目的＝毎レースでなく年数回の大妙味G1を逃さず厚く回収（実績: 2025天皇賞秋5万→25万、有馬10万→60万）。「勝負か見送りか／勝負ならいくら張るか」は bet.html で判定。**
1. **大妙味＝較正した自前pと市場価格の乖離(エッジ)が大きいスポット**。bet.htmlで各馬のブレンドp・エッジ・EVを出し、**最大EV≥しきい値(既定1.25)のレースだけ"勝負"、他は見送り**（年数回に集中）。
2. **海外のFL bias数値は日本に転用不可**（米国の大穴-61%等は香港・日本で成立弱い: Busche 1988/94）。「人気薄を避ける/狙う」を一律ルール化しない＝自前較正でレースごとに判断。
3. **JRA固有スポット**：①**強い本命の複勝/連は市場が過小評価しがち**（Ashiya 2010、古いデータ＝方向の参考）＝本命は飛ばすより複勝・連の軸に。②**過剰人気は確率の誤認知で系統的**（有名騎手/前走人気/路線人気で売れすぎ＝EVマイナス）＝消しの相手に。
4. **【最重要シグナル】締切直前(最後5分)の人気急騰馬は実現リターン高い**（JRA-VAN89万頭, δ=-0.3559）＝賢い金が直前に入るサイン。bet.html③に30分前/直前オッズを入れて検知（シグナルであって必勝でない・単勝のみ・純益未証明）。
5. **券種＝歪みの取りやすさで選ぶ**：**本線は馬連・3連複**（単勝からの推定精度高: Harville/Lo&Busche）。**馬単・3連単は推定劣化＝跳ね用に少額**（「3連単が最大妙味」説は検証で棄却）。**複勝・ワイドは強い本命の軸固め**（分散小）。組み立て＝①複複/ワイドで土台→②馬連/3連複でエッジ馬を本線(厚め)→③3連単は上振れ少額。
6. **サイジング＝fractional Kelly（blendへのフルKellyと数学的に等価: AAMAS2012）**。エッジ大ほど厚く・推定不確かなほどshrink（Metel2018/Baker&McHale2013）。**1レースは資金の20%以内目安**。bet.htmlがconviction別の推奨単勝換算額を自動算出（既定λ=0.5・w=0.4）。
7. **検証＝CLV（クロージングライン超過）が自前pの優位を測る最良級指標**。JRAはpari-mutuelで確定オッズが締切後も動くため、締切直前動態(項4)で代替。勝負レースのエッジ・結果・CLVを track.html に蓄積し長期で巧拙を判定。

## 11. 二勘定の運用ルール（勝負枠 / 回収枠・bet.htmlに実装）
**利益エンジン＝大妙味レースを厚く（勝負枠）。見送り見込みレースは"薄く・別勘定"で回収型（回収枠）＝赤字前提の娯楽/練習/データ枠。両者を物理的に分け、回収枠が勝負枠を侵食しないようにする。**
1. **勝負枠（利益）**：bet.htmlで最大EV≥しきい値(1.25)のレースのみ。馬連/3連複を本線に厚く＋3連単跳ね。conviction別 half-Kelly、**1レース資金の〜20%まで**。
2. **回収枠（娯楽/練習/データ）**：見送り見込みレースで薄く。**本命の複勝＋本命軸ワイドの傾斜配分**（JRAで唯一市場が過小評価しがちな"強い本命の複勝/連"を突く＝同じ-EVでも傷が浅く分散小）。**3連複の広げすぎはトリガミ＝避ける**。**1レース上限（既定¥5,000）＋月上限（既定¥30,000）を厳守**。月上限超なら今月は薄張りも止め勝負枠を温存。
3. **位置づけを誤認しない**：回収型は分散を下げるだけで期待値の符号は変えない＝長期は控除率分マイナス。回収枠は**利益源でなく赤字前提の参加枠**。profitは勝負枠が生む。
4. **★は事前のprior、決定は当日のbet.html**：autumn-watch.htmlの★は見込みにすぎない。**低★でも当日EV≥しきい値が出たら勝負枠へ格上げ**、★★★でも当日エッジが消えていれば回収枠へ降格 or 見送り。最終判断は常に当日の較正EV。
5. **bet.htmlの回収モード**：見送り判定時に本命(混p最上位)軸の複勝/ワイドの抑えめ買い目を上限内で自動提案。本命もEV0.85未満なら最小額 or 完全見送りを促す。

## 12. 自分p生成の標準（8視点ウェイト・score.html）
**自分pは8視点を重み付け加重して機械的に生成する（敗因＝データ細部偏重・地力軽視を重みで矯正）。** score.htmlが本体。予想を組む時もこの重み感覚で印→pを付ける。
1. **推奨ウェイト（合計100%）**：①展開12・②血統12・**③近走22(★地力)**・④コース馬場14・⑤オッズ6(↓低)・⑥調教10・**⑦指数18(★地力)**・⑧データ傾向6(↓低)。**地力(近走/指数)で40%**、薄い⑧データ傾向と市場由来の⑤オッズは各6%に抑える。
2. **⑤オッズを低重みにする理由**：自分pは"市場から独立した地力評価"であるべき。市場はbet.htmlのブレンドで別途反映するので、ここで重く入れると二重計上になる。
3. **採点**：各馬を視点ごと1〜5点（3=市場並み/平均、5=その視点で明確に上位、1=明確に劣る）。多くは3のままで、突出/劣勢の視点だけ動かす。
4. **p化**：加重平均スコアS(1〜5)→ softmax(β·S) で正規化。βは「確率のメリハリ」（既定0.90、大=上位集中）。出力pを印(◎○▲△=p順位)とともに表示。
5. **連携**：score.htmlの「勝負判定へ送る」でlocalStorage経由 bet.html に自分p・オッズを渡し、市場ブレンドEV→勝負/見送り→サイジングへ。フロー＝autumn-watch(★事前ふるい)→score(自分p)→bet(EV/勝負/サイジング)→track(較正)。
